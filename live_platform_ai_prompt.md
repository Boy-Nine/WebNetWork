# Web 版直播平台项目需求扩展与 AI 生成提示词

> 目标：基于“用户 / 主播两种账号类型、用户一键开播、贝壳打赏、余额不足充值、充值升级等级机制”的初始想法，扩展成一份可直接交给 AI 生成项目的详细需求文档。

---

## 1. 项目定位

我要做一个 Web 版直播平台，类似轻量版直播社区。平台支持普通用户观看直播、打赏礼物、充值贝壳、升级等级；同时用户也可以一键开播，成为主播进行直播互动。

平台核心玩法：

- 用户进入平台后可以浏览直播间列表。
- 用户可以进入直播间观看主播直播。
- 用户可以发送弹幕 / 聊天消息。
- 用户可以使用平台虚拟货币“贝壳”购买礼物并打赏主播。
- 用户贝壳不足时，需要提示充值。
- 用户充值贝壳后获得成长值 / 经验值，并根据累计充值或成长值升级用户等级。
- 用户可以点击“一键开播”，快速创建自己的直播间。
- 主播可以看到直播间在线人数、礼物流水、聊天消息、直播收益等数据。
- 管理后台可以管理用户、主播、直播间、礼物、充值订单、打赏记录等。

## 2. 账号体系

### 2.1 账号类型

平台默认有两种账号身份：

1. **普通用户**
   - 可以注册、登录。
   - 可以浏览直播间。
   - 可以进入直播间观看直播。
   - 可以发送聊天消息。
   - 可以充值贝壳。
   - 可以打赏礼物。
   - 可以查看自己的余额、等级、充值记录、打赏记录。
   - 可以点击“一键开播”，创建直播间。

2. **主播**
   - 主播本质上也是用户，只是处于开播状态时拥有主播能力。
   - 可以创建直播间。
   - 可以设置直播标题、封面、分类、公告。
   - 可以开始直播、暂停直播、结束直播。
   - 可以查看本场直播数据，例如在线人数、礼物收入、弹幕数量、观看人数。
   - 可以查看历史直播记录和收益记录。

> 建议设计：不要把“用户”和“主播”做成完全割裂的两个表。建议所有账号统一存在 `users` 表中，用户点击开播后生成 `live_rooms` 和 `live_sessions` 数据。这样每个用户都可以成为主播。

### 2.2 用户字段建议

用户信息包含：

- 用户 ID
- 昵称
- 头像
- 手机号 / 邮箱
- 密码哈希
- 账号状态：正常、禁用、注销
- 用户等级
- 当前贝壳余额
- 累计充值金额
- 累计成长值
- 是否正在直播
- 创建时间
- 更新时间

## 3. 虚拟货币：贝壳

### 3.1 贝壳定义

贝壳是平台内部虚拟货币，用于购买礼物并打赏主播。

建议规则：

- 1 元人民币 = 10 贝壳
- 贝壳只能通过充值获得
- 贝壳不可提现
- 贝壳用于购买礼物
- 用户打赏礼物后，扣除对应贝壳
- 主播获得礼物收益，平台可抽成

### 3.2 贝壳余额规则

用户账户中需要维护：

- 当前贝壳余额
- 累计充值贝壳
- 累计消费贝壳
- 冻结贝壳，可选

所有贝壳变动必须记录流水，不能只改余额。

贝壳流水类型：

- 充值增加
- 打赏扣除
- 后台赠送
- 订单退款
- 活动奖励
- 系统修正

### 3.3 余额不足提示

当用户点击礼物打赏时：

1. 检查用户当前贝壳余额。
2. 如果余额 >= 礼物价格，则扣除贝壳并生成打赏记录。
3. 如果余额 < 礼物价格，则弹出充值提示：
   - 当前余额不足
   - 需要多少贝壳
   - 当前余额多少
   - 跳转充值按钮
   - 取消按钮

示例提示文案：

> 贝壳余额不足，当前余额 20 贝壳，赠送该礼物需要 99 贝壳。请先充值后继续打赏。

## 4. 礼物系统

### 4.1 礼物类型

礼物可以分为：

- 普通礼物：价格低，例如小贝壳、爱心、鲜花
- 高级礼物：价格中等，例如跑车、游艇、皇冠
- 豪华礼物：价格高，例如城堡、星河、宇宙飞船
- 连击礼物：支持连续点击，显示连击动画

### 4.2 礼物字段

每个礼物包含：

- 礼物 ID
- 礼物名称
- 礼物图标
- 礼物动画，可选
- 礼物价格，单位贝壳
- 礼物排序
- 是否上架
- 创建时间
- 更新时间

### 4.3 打赏流程

用户在直播间中：

1. 打开礼物面板。
2. 选择礼物。
3. 点击赠送。
4. 前端调用打赏 API。
5. 后端检查用户登录状态。
6. 后端检查直播间是否正在直播。
7. 后端检查贝壳余额是否足够。
8. 余额足够则扣除贝壳。
9. 生成礼物打赏记录。
10. 生成贝壳消费流水。
11. 更新主播本场收益。
12. 通过 WebSocket 广播礼物消息。
13. 直播间显示礼物动画和聊天提示。

### 4.4 打赏消息示例

直播间消息示例：

> 用户 小明 送给主播 小鱼 一个 跑车，价值 999 贝壳。

## 5. 用户等级机制

### 5.1 等级设计

用户等级根据累计充值金额或累计成长值计算。

建议采用成长值：

- 每充值 1 元 = 10 成长值
- 用户累计成长值达到等级门槛后自动升级
- 等级不会因为消费贝壳而下降

### 5.2 等级表建议

| 等级 | 名称 | 所需累计充值 | 所需成长值 | 权益 |
|---|---|---:|---:|---|
| Lv1 | 新人 | 0 元 | 0 | 基础聊天、观看直播 |
| Lv2 | 贝壳会员 | 10 元 | 100 | 等级标识 |
| Lv3 | 银贝会员 | 50 元 | 500 | 彩色昵称 |
| Lv4 | 金贝会员 | 100 元 | 1000 | 进场提示 |
| Lv5 | 钻石会员 | 500 元 | 5000 | 专属头像框 |
| Lv6 | 王者会员 | 1000 元 | 10000 | 高亮弹幕 |
| Lv7 | 至尊会员 | 5000 元 | 50000 | 专属礼物特效 |

### 5.3 升级逻辑

用户充值成功后：

1. 增加贝壳余额。
2. 增加累计充值金额。
3. 增加成长值。
4. 根据成长值匹配最高等级。
5. 如果等级提升，则更新用户等级。
6. 生成升级通知。

升级提示示例：

> 恭喜你升级到 Lv4 金贝会员，已解锁进场特效。

## 6. 充值系统

### 6.1 充值套餐

建议提供固定套餐：

| 套餐 ID | 金额 | 贝壳数量 | 赠送贝壳 | 总贝壳 |
|---|---:|---:|---:|---:|
| 1 | 6 元 | 60 | 0 | 60 |
| 2 | 30 元 | 300 | 20 | 320 |
| 3 | 68 元 | 680 | 80 | 760 |
| 4 | 128 元 | 1280 | 180 | 1460 |
| 5 | 328 元 | 3280 | 520 | 3800 |
| 6 | 648 元 | 6480 | 1200 | 7680 |

### 6.2 充值流程

1. 用户进入充值页面。
2. 选择充值套餐。
3. 创建充值订单。
4. 选择支付方式。
5. 模拟支付 / 接入真实支付。
6. 支付成功后回调后端。
7. 后端校验订单状态。
8. 给用户增加贝壳。
9. 写入贝壳流水。
10. 更新累计充值金额和成长值。
11. 判断是否升级。
12. 返回充值成功页面。

### 6.3 MVP 阶段支付建议

如果是项目初版，不一定要真实接入微信 / 支付宝，可以先做模拟支付：

- 创建充值订单后跳转到模拟支付页。
- 点击“确认支付”后将订单状态改为已支付。
- 后端完成加贝壳和升级逻辑。
- 后续再替换成真实支付网关。

## 7. 直播间系统

### 7.1 直播间核心功能

直播间需要包含：

- 直播标题
- 主播信息
- 直播封面
- 在线观看人数
- 聊天区域
- 礼物面板
- 礼物动画区域
- 用户榜单
- 主播收益
- 关注按钮，可选
- 分享按钮，可选

### 7.2 开播流程

用户点击“一键开播”：

1. 检查用户是否登录。
2. 检查用户账号是否正常。
3. 检查用户是否已经在直播中。
4. 如果没有直播间，则创建直播间。
5. 创建本次直播场次 `live_session`。
6. 设置直播状态为 live。
7. 返回推流地址 / WebRTC 房间信息 / 模拟直播地址。
8. 前端跳转到主播工作台。

### 7.3 直播状态

直播间状态建议：

- `created`：已创建，未开播
- `live`：直播中
- `paused`：暂停中
- `ended`：已结束
- `banned`：被封禁

### 7.4 MVP 阶段直播方案

如果暂时不做真实音视频推流，可以先做“模拟直播”：

- 主播进入直播间后显示摄像头占位区域。
- 可以调用浏览器摄像头 `getUserMedia` 做本地预览。
- 观众端显示主播封面或模拟视频区域。
- 聊天、礼物、在线人数、榜单等互动功能先完整实现。

后续升级为真实直播：

- WebRTC：适合低延迟互动直播。
- HLS：适合大规模观看，但延迟较高。
- 第三方云直播：腾讯云、阿里云、声网、七牛云等。

## 12. API 接口建议

### 12.1 认证接口

- `POST /api/auth/register` 注册
- `POST /api/auth/login` 登录
- `POST /api/auth/logout` 退出登录
- `GET /api/auth/me` 获取当前用户信息

### 12.2 用户接口

- `GET /api/users/me` 获取个人资料
- `PUT /api/users/me` 修改个人资料
- `GET /api/users/me/wallet` 获取贝壳余额
- `GET /api/users/me/level` 获取等级信息
- `GET /api/users/me/transactions` 获取贝壳流水

### 12.3 直播接口

- `GET /api/live/rooms` 获取直播间列表
- `GET /api/live/rooms/:id` 获取直播间详情
- `POST /api/live/start` 一键开播
- `POST /api/live/rooms/:id/end` 结束直播
- `POST /api/live/rooms/:id/join` 进入直播间
- `POST /api/live/rooms/:id/leave` 离开直播间
- `GET /api/live/rooms/:id/messages` 获取聊天历史

### 12.4 礼物接口

- `GET /api/gifts` 获取礼物列表
- `POST /api/live/rooms/:id/send-gift` 发送礼物

请求示例：

```json
{
  "giftId": 1,
  "quantity": 1
}
```

余额不足响应示例：

```json
{
  "code": "INSUFFICIENT_SHELLS",
  "message": "贝壳余额不足，请先充值",
  "data": {
    "currentBalance": 20,
    "requiredShells": 99,
    "shortage": 79
  }
}
```

### 12.5 充值接口

- `GET /api/recharge/packages` 获取充值套餐
- `POST /api/recharge/orders` 创建充值订单
- `POST /api/recharge/orders/:id/mock-pay` 模拟支付成功
- `GET /api/recharge/orders` 获取充值记录

### 12.6 管理后台接口

- `GET /api/admin/dashboard` 后台看板
- `GET /api/admin/users` 用户列表
- `PATCH /api/admin/users/:id/status` 修改用户状态
- `GET /api/admin/live/rooms` 直播间管理
- `PATCH /api/admin/live/rooms/:id/ban` 封禁直播间
- `GET /api/admin/gifts` 礼物管理
- `POST /api/admin/gifts` 新增礼物
- `PUT /api/admin/gifts/:id` 修改礼物
- `GET /api/admin/recharge-orders` 充值订单
- `GET /api/admin/gift-records` 打赏记录

## 13. 技术栈建议

### 13.1 推荐技术栈一：全栈 TypeScript

适合让 AI 快速生成完整项目。

- 前端：Next.js / React / TypeScript
- UI：Tailwind CSS + shadcn/ui
- 后端：Next.js API Routes 或 NestJS
- 数据库：PostgreSQL
- ORM：Prisma
